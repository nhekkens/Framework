Introduction

After working on a few reskins and projects that had already been started by another developer I came across some areas that I felt could be refined. Refining these areas, although it will take a little more time and thought in the initial build, will save time on maintaining the project. This will be particularly helpful if and when the design/layout changes after development has started.

Establishing support

If it is possible, establish what browser support is required for the project. One common issue we have is aligning content blocks that sit in the same row but have varying heights on content. The default is to use display: block; and fix heights which is problematic as you have use character limits. If we are only supporting the latest version of IE then we can switch to using display: flex; which, as the name suggest, is much more flexible. Flexbox gives us much more control over alignment without the need to fix heights or use character limits. Find out more about flexbox.

Design review and BEM

It's tricky as we often start a project before we have signed off designs but an adjustment in approach to styles will help going forward. Particularly if the design/layout changes after development has started.

Going through all designs available and breaking each one down into components will help with the BEM approach. We can stick to the components structure we have in place but be stricter on how we name classes and the way we write the styles. The key to keeping maintenance quick and easy is not to nest selectors.

Buttons are a good example...

<button class="button">
    Normal button
</button>
<button class="button button--state-success">
    Success button
</button>
<button class="button button--state-danger">
    Danger button
</button>

.button {
    background-color: #fff;
    border: 2px solid $brand-primary;
    border-radius: $border-radius-base;
    margin-top: $spc;
    padding: floor($spc / 2) floor($spc * 2);

    &:hover {
        background-color: $brand-primary;
        color: #fff;
    }
}
.button--state-success {
    border-color: $brand-success;

    &:hover {
        background-color: $brand-success;
        color: #fff;
    }
}
.button--state-danger {
    &:hover {
        background-color: $brand-danger;
    }
}
